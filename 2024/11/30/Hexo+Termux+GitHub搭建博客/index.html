Hexo+Termux+GitHub搭建博客

发表于2024-11-30|更新于2024-12-11|教程
|总字数:490|阅读时长:1分钟|浏览量:|评论数:

全程开启魔法避免错误

  1. 安装Termux和必要的工具首先，你需要在你的Android设备上安装Termux应用。然后，在Termux中执行以下命令来更新软件包列表并安装Node.js、Git等必要的工具：
    pkg update && pkg upgrade -y
    pkg install nodejs-lts vim git -y

  2. 安装Hexo
    npm install hexo-cli -g

创建hexo文件夹
mkdir hexo

给予完全访问权限
chmod 777 hexo

切换到hexo目录
cd hexo

初始化项目
hexo init

请下载安装 pug 以及 stylus 的渲染器：
npm install hexo-renderer-pug hexo-renderer-stylus –save

生成静态页面
hexo g

  1. 启动本地服务器预览博客在Hexo目录下启动本地服务器：
    hexo s
    访问 http://localhost:4000 来预览你的博客。

  2. 部署到GitHub
    进入GitHub创建一个仓库为 你的用户名+github.io

配置必要信息
git config –global user.name “放你的Github用户名”
git config –global user.email “放你的Github登录邮箱”

安装openssh
pkg install openssh

生成连接密钥(一直回车)
ssh-keygen -t rsa -C “放你的Github登录邮箱”
根据路径(.ssh/id_rsa.pub)找到这个文件，然后复制里面的全部内容

连接SSH，进入GitHub→Settings→SSH and GPG keys→New SSH key→Title 随便输入，Key 粘贴刚刚复制的内容，不要带有任何字符，点击Add SSH key
创建密钥，进入GitHub→Settings→Developer settings→Personal access tokens→Note 随便填→Expiration设置成永不→下面权限全勾上→Generate token(生成密钥后要记住，这只显示一次)

进入hexo目录_config.yml文件，滑到最下面，然后修改
deploy:
type: git
repo: https://github.com/你的GitHub用户名/你的GitHub用户名+.github.io.git
broanch: main

这已经大功告成了
每次上传到github需要输入以下命令
hexo clean 清理生成的静态文件
hexo g 生成静态文件
hexo d 上传到GitHub
一条一条太麻烦了，直接一条龙服务命令
hexo clean && hexo g && hexo d
